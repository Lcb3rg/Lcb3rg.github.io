用户与组管理：

  1. 用户概述

    1. 每个用户登录系统后，拥有不同的操作权限
    2. 每个账户有自己唯一的SID（安全标识符）
    3. Windows系统管理员administrator的UID是500，普通用户的UID是1000开始
    4. 不同的账户拥有不同的权限，为不同的账户赋权限，也就是为不同账户的SID赋权限
    5. 查看SID值：whoami /user
    6. 账户密码存储位置：c:\windows\system32\config\SAM #暴力破解#撞库
    7. Windows系统上，默认密码最长有效期42天

  2. 内置账户

    1. 给人使用的账户：
      • administrator #管理员账户
      • guest #来宾账户
    2. 计算机服务组件相关的系统账户：
      • system #系统账户==权限至高无上
      • local services #本地服务账户==权限等于普通用户
      • network services #网络服务账户==权限等于普通用户

  3. 配置文件

    1. 每个用户都有自己的配置文件（家目录），在用户第一次登录时自动产生
    2. 路径是：win7/win2008–>c:\用户\
      • xp/win2003–>c:\Documents and Settings\

  4. 用户管理命令

    1. net user #查看用户列表
    2. net user 用户名 密码 #改密码
    3. net user 用户名 密码 /add #创建一个新用户
    4. net user 用户名 /del #删除一个用户
    5. net user 用户名 /active:yes/no #激活或禁用账户

  5. 组概述

    1. 组的作用：简化权限的赋予
    2. 赋权限的方式：
      • 用户–组–赋权限
      • 用户–赋权限

  6. 内置组

    1. 内置组的权限默认已经被系统赋予
    2. administrators #管理员组
    3. guests #来宾组
    4. users #普通用户组，默认新建用户都属于该组
    5. network #网络配置组
    6. print #打印机组
    7. Remote Desktop #远程桌面组

  7. 组管理命令

    1. net localgroup #查看组列表
    2. net localgroup 组名 #查看该组的成员
    3. net localgroup 组名 /add #创建一个新的组
    4. net localgroup 组名 用户名 /add #添加用户到组
    5. net localgroup 组名 用户名 /del #从组中踢除用户
    6. net localgroup 组名 /del #删除组

服务器远程管理：

  1. netstat -an #查看本机开放的所有端口

    • telnet(远程登录)：23
    • 远程桌面协议（RDP）：3389
    • 文件传输：445

  2. 远程管理类型

    • 远程桌面（图形）
    • telnet（命令行）

NTFS安全权限：

  1. NTFS权限概述

    1. 通过设置NTFS权限，实现不同的用户访问不同的权限（针对文件/文件夹）
    2. 分配了正确的访问权限后，用户才能访问其资源
    3. 设置权限防止资源被篡改、删除

  2. 文件系统概述

    1. 文件系统即在外部存储设备上组织文件的方法
    2. 常见的文件系统：
      • FAT Windows
      • NTFS Windows
      • EXT Linux常见

  3. NTFS文件系统特点

    1. 提高磁盘读写性能
    2. 可靠性
      • 加密文件系统
      • 访问控制列表（设置权限）（ACL）
    3. 磁盘利用率
      • 压缩
      • 磁盘配额
    4. 支持单个文件大于4个G

  4. 修改NTFS权限

    1. 取消权限继承

      • 作用：取消后，可以任意修改权限列表
      • 方法：文件夹右键属性—安全—高级—去掉第一个对号—选择复制即可

    2. 文件及文件夹权限

    3. 权限累加

      • 当用户同时属于多个组时，权限是累加的

    4. 拒绝最大

      • 当用户权限累加时，如遇到拒绝权限，拒绝最大

    5. 取得所有权

      • 默认只有administrator有这个权限
      • 作用：可以将任何文件夹的所有者改为administrator

    6. 强制继承

      • 作用：对下强制继承父子关系
      • 方法：文件夹右键属性–安全–高级–勾上第二个对号，即可

    7. 文件复制对权限的影响

      • 文件复制后，文件的权限会被目标文件夹的权限覆盖

文件共享服务器：（cifs）

  1. 共享服务器概述

    1. 通过网络提供文件共享服务，提供文件下载和上传

  2. 创建共享

    1. 方法：文件夹右键属性–共享–开启共享–设置共享名–设置共享权限
    2. 注意：
      • 在本地登录时，只受NTFS权限影响
      • 在远程登录时，将受共享和NTFS权限的共同影响，且去交集
      • 所以建议设置共享权限为everyone完全控制，然后具体的权限需求在NTFS权限中设置即可

  3. 访问共享

    1. 在开始运行或我的电脑地址栏中，输入UNC地址：
    2. \文件共享服务器IP
    3. \文件共享服务器IP\共享名

  4. 创建隐藏的共享

    1. 方法：共享名$

  5. 访问隐藏共享的方法

    1. \服务器IP\共享名$

  6. 共享相关命令

    1. net share #列出共享列表
    2. net share 共享名 /del #删除共享
    3. whoami #显示当前用户的名字
    4. whoami /user #显示当前用户信息

  7. 屏蔽系统隐藏共享自动产生

    1. 打开注册表

      1. 打开注册表编译器：regedit

    2. 定位共享注册表位置

  8. 查看本地网络连接状态

    1. netstat -an

  9. 关闭445服务

    1. 445端口是文件传输端口–>SMB（网络通信协议）
    2. 方法1：cmd–>services.msc，并停止及禁用server服务
    3. 方法2：禁止被访问445，配置高级安全防火墙–>入站规则
